Retail shops carry dozens of depots for storage and they summary stock management, real-time updates, market analysis, marketing and the management of human resource etc. Our application developed for performing these jobs can make them a lot simpler and easy to use. Tracking of your employees helps you study your business better and to improve your production. There are two interfaces for this solution.

Web Interface - Allows the administrator to monitor the overall working of the system.

Mobile interface - Provided to the employees to make real-time updates of their daily works. This application comes handy at the time of monthly inspections. It helps you to analyse the stocks, sales and production status of the system.

Features

Tracking : The employees can login using their Smartphone before starting their work for the day. The login details include login time, location tracking using GPS. While from office, the administrator can track their employees from their computer by searching them according to their corresponding depots.

Depot : Using the application from their smart phones, the employees can perform the following features.

Form : The employees can type the corresponding depot code on their android mobiles and a form which includes the stocks details.

Receipts : This application helps you to know from where each of the stocks came from and to whom they were delivered to. They can submit these details in real-time and these information’s could be received at the office instantly.

Sales : Here you can update to whom the stocks were delivered to, at what time and in what quantity and so on. A standard form has to be filled from their smart phones which will be updated at the office computer instantly after submission from the other end. From the office, the administrator can check the stocks at each depot, check daily stocks and receipts. There are three types of search option available to him:

Search based on depots
Search based on depot and date, and
Search based on outlets.

Outlets : The outlets are categorized into three, namely beverages, consumer fed and the bars. There are 18 employees, one each from the 18 different depots who are in charge of the different outlets across the state. Each of the employees will be given a Smartphone in which they can enter a unique code for each outlets and enter the required details into it. Each unique code corresponds to each outlets and the requirements and updates of each of the outlets can be updated and send to the office instantly from their smart phones.

Server side
The server side software has a control panel in which the admin can login through a username and password. The software contains mainly 3 sections Admin, depot and outlet which consist of activities in each section.

Admin
The admin section contains mainly 6 activities

1. Add Depot
This is the first activity of the admin in which the admin can dynamically add a depot. The depot detail includes a depot name and a depot code. The admin can view the entered depots as a table which includes the depot id, depot name and the depot code. The admin can view the profile of each depot from the list. He also has the provision to delete a depot from the list.

2. Add Executives
In this section the admin can add an Executive (employee) dynamically. Admin can enter the executive’s personal and official details in separate forms. A personal detail includes name, address, location, mobile number and mail Id. The Executive’s official detail includes login user name, password, a depot assigned to that executive and a device Id. The admin can view the registered employee details which includes executive name, depot code, executive username and password. The admin can view the complete profile of each executive separately. Admin can have a calendar view of each executive from their profile which includes login time, login location, day and all other activities.

3. Add Category
In this section the admin can add the category name and type dynamically. He can view the entered category from the table.

4. Add Outlets
In this section the admin can dynamically add number of outlets under a depot. He can add outlet with an outlet Id and name by selecting the depot name and category. There may be a number of outlets under a depot. The admin can view the entered outlets based on the category and their counts. He can view list from the table which gives an outlet Id, outlet code and outlet name entered under a category.

5. Add products
In this activity the admin can add the product details dynamically. The product details include product name, product code, product type and the company name. Admin can view the entered products from the list. He can also edit or delete a product from the list.

6. Add product type
In this activity the admin can enter the product types and view from the list. Product type can be any liquor. These are the main admin activities in the section

Depot
This section contains mainly 5 activities

1. Daily Closing stocks
In this activity the admin can view the daily closing stocks at the depot. He can retrieve the data of the corresponding month and year. The data viewed as a table with date, depot name and the product description. The admin can also view the executive who has reported from the table. Thus he can retrieve the closing stocks of every month.

2. Daily sales
In this section the admin can view the daily sales at the depot. The admin can view the daily sale activities of the month. The data includes date, depot, type, product descriptions, and the executive who reported it. The admin can also edit or delete from the sale entry.

3. Current Stocks
In this section the admin can view the current stock at each depot. He can get a table view of each depot. It includes the product name, type and total available stock at the depot.

4. Receipt Report
In this section the admin can get the receipt report at the depot. The admin can retrieve the monthly data of goods received sheet at each depot. The receipt report includes the date, depot name, GRS number, supplier, permit number, products, their arrived date, upheld date and the executive who reported it.

5. Necessity report
In this section the admin can view the daily necessity of the products at a depot. The admin can retrieve the necessity details which include date, depot, product descriptions, their total necessity, and executives who reported it.

Outlet
This section has mainly 5 activities

1. Monthly Opening stock
In this activity the admin can view the monthly opening stock at the outlet. He can retrieve the data by giving the depot name, category, and month. The admin can view the opening stock at the outlet which includes outlet name, product descriptions and executive who reported it. The admin can edit or delete the monthly opening stock and monthly closing stock. In this activity the admin can view the monthly closing stock at the outlet. He can retrieve the data by giving the depot name, category, and month. The admin can view the closing stock at the outlet which includes outlet name, product descriptions and executive who reported it.

2. Monthly closing stock
In this activity the admin can view the monthly closing stock at the outlet. He can retrieve the data by giving the depot name, category, and month. The admin can view the closing stock at the outlet which includes outlet name, product descriptions and executive who reported it. The admin can edit or delete the monthly closing stock entry.

3. Receipt entry
In this section the admin can get the daily receipt entry of the outlets. He can retrieve the data by giving the depot name, category, and month. The admin can view the receipt at the outlet.

4. Stock at visit
In this section the admin can get the daily stock at visit of the outlets. He can retrieve the data by giving the depot name, category, and month. Admin can view the stock at visit which includes date, outlet name, product descriptions and executive who reported it. . The admin can edit or delete the stock at visit entry

5. Necessity Entry
In this section the admin can get the daily necessity entry of the outlets. He can retrieve the data by giving the depot name, category, and month. Admin can view the necessity entry which includes date, outlet name, product descriptions and executive who reported it. The admin can edit or delete the necessity entry. These are the main sections which includes in the server side of the software.
